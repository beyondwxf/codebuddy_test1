## 产品概述

为若依风格后台管理系统添加完整的商品管理模块，支持商品的增删改查、库存管理、多供应商管理等功能。

## 核心功能

- **仓库管理**: 管理多个仓库信息（名称、编码、省份、地址、类型、状态）
- **商品管理**: 管理商品基本信息（名称、SKU编码、简介、预览图、移动端宣传页、售价、进货价、单位、状态）
- **商品-供应商关联**: 支持同一商品由多个供应商供货，记录供应商商品编码、报价、交货周期
- **商品库存管理**: 管理商品在各个仓库的库存数量和预警数量
- **富文本编辑**: 移动端产品宣传页支持富文本编辑
- **导入导出**: 支持Excel数据导入导出
- **权限控制**: 集成若依权限体系，支持按钮级权限控制

## 技术栈

- 后端: Spring Boot 3.2.0 + MyBatis-Plus 3.5.5 + Java 17 + EasyExcel
- 前端: Vue 3 + Element Plus + Vue Router 4 + Vite
- 数据库: MySQL 8.0 + Redis
- 富文本编辑器: 使用 `@wangeditor/editor` 和 `@wangeditor/editor-for-vue`

## 实现方案

### 数据库设计

#### 1. 仓库表 (wms_warehouse)

```sql
CREATE TABLE wms_warehouse (
    warehouse_id     BIGINT        NOT NULL AUTO_INCREMENT COMMENT '仓库ID',
    warehouse_name   VARCHAR(100)  NOT NULL COMMENT '仓库名称',
    warehouse_code   VARCHAR(50)  NOT NULL COMMENT '仓库编码',
    province         VARCHAR(50)   DEFAULT NULL COMMENT '省份',
    address          VARCHAR(500)  DEFAULT NULL COMMENT '详细地址',
    warehouse_type   VARCHAR(20)   DEFAULT '0' COMMENT '仓库类型（0自有 1租赁）',
    status           CHAR(1)       DEFAULT '0' COMMENT '状态（0正常 1停用）',
    del_flag         CHAR(1)       DEFAULT '0' COMMENT '删除标志（0存在 2删除）',
    create_by        BIGINT        DEFAULT NULL COMMENT '创建者',
    create_time      DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by        BIGINT        DEFAULT NULL COMMENT '更新者',
    update_time      DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    remark           VARCHAR(500)  DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (warehouse_id),
    UNIQUE KEY uk_warehouse_code (warehouse_code)
) ENGINE=InnoDB AUTO_INCREMENT=100 COMMENT='仓库信息表';
```

#### 2. 商品表 (wms_product)

```sql
CREATE TABLE wms_product (
    product_id        BIGINT         NOT NULL AUTO_INCREMENT COMMENT '商品ID',
    product_name      VARCHAR(200)   NOT NULL COMMENT '商品名称',
    sku_code          VARCHAR(100)   NOT NULL COMMENT 'SKU编码',
    description       VARCHAR(1000)  DEFAULT NULL COMMENT '商品简介',
    preview_image     VARCHAR(500)   DEFAULT NULL COMMENT '预览图URL',
    mobile_content    LONGTEXT       DEFAULT NULL COMMENT '移动端宣传页内容（富文本）',
    sale_price        DECIMAL(10,2)  DEFAULT 0.00 COMMENT '售价',
    purchase_price    DECIMAL(10,2)  DEFAULT 0.00 COMMENT '进货价',
    unit              VARCHAR(20)    DEFAULT '个' COMMENT '单位',
    status            CHAR(1)        DEFAULT '0' COMMENT '状态（0上架 1下架）',
    del_flag          CHAR(1)        DEFAULT '0' COMMENT '删除标志（0存在 2删除）',
    create_by         BIGINT         DEFAULT NULL COMMENT '创建者',
    create_time       DATETIME       DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by         BIGINT         DEFAULT NULL COMMENT '更新者',
    update_time       DATETIME       DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    remark            VARCHAR(500)   DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (product_id),
    UNIQUE KEY uk_sku_code (sku_code)
) ENGINE=InnoDB AUTO_INCREMENT=100 COMMENT='商品信息表';
```

#### 3. 商品供应商关联表 (wms_product_supplier)

```sql
CREATE TABLE wms_product_supplier (
    id                  BIGINT      NOT NULL AUTO_INCREMENT COMMENT 'ID',
    product_id          BIGINT      NOT NULL COMMENT '商品ID',
    supplier_id         BIGINT      NOT NULL COMMENT '供应商ID',
    supplier_sku_code   VARCHAR(100) DEFAULT NULL COMMENT '供应商商品编码',
    supplier_price      DECIMAL(10,2) DEFAULT 0.00 COMMENT '供应商报价',
    lead_time           INT         DEFAULT NULL COMMENT '交货周期（天）',
    is_default          CHAR(1)      DEFAULT '0' COMMENT '是否默认供应商（0否 1是）',
    del_flag            CHAR(1)      DEFAULT '0' COMMENT '删除标志（0存在 2删除）',
    create_by           BIGINT      DEFAULT NULL COMMENT '创建者',
    create_time         DATETIME    DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by           BIGINT      DEFAULT NULL COMMENT '更新者',
    update_time         DATETIME    DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    remark              VARCHAR(500) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id),
    UNIQUE KEY uk_product_supplier (product_id, supplier_id)
) ENGINE=InnoDB AUTO_INCREMENT=100 COMMENT='商品供应商关联表';
```

#### 4. 商品库存表 (wms_product_inventory)

```sql
CREATE TABLE wms_product_inventory (
    id                BIGINT      NOT NULL AUTO_INCREMENT COMMENT 'ID',
    product_id        BIGINT      NOT NULL COMMENT '商品ID',
    warehouse_id      BIGINT      NOT NULL COMMENT '仓库ID',
    stock_quantity    INT         DEFAULT 0 COMMENT '库存数量',
    warning_quantity  INT         DEFAULT 0 COMMENT '预警数量',
    del_flag          CHAR(1)      DEFAULT '0' COMMENT '删除标志（0存在 2删除）',
    create_by         BIGINT      DEFAULT NULL COMMENT '创建者',
    create_time       DATETIME    DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by         BIGINT      DEFAULT NULL COMMENT '更新者',
    update_time       DATETIME    DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    remark            VARCHAR(500) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id),
    UNIQUE KEY uk_product_warehouse (product_id, warehouse_id)
) ENGINE=InnoDB AUTO_INCREMENT=100 COMMENT='商品库存表';
```

## 实施细节

### 后端实施要点

1. **遵循现有模式**: 参照SupplierController的实现方式，使用MyBatis-Plus的BaseMapper
2. **逻辑删除**: 所有表使用del_flag字段（'0'存在，'2'删除）
3. **权限控制**: 使用@PreAuthorize注解，权限字符串格式：wms:warehouse:list、wms:product:list
4. **统一响应**: 使用Result<T>和TableDataInfo<T>封装响应
5. **操作日志**: 使用@Log注解记录操作日志
6. **Excel处理**: 使用EasyExcel实现导入导出功能

### 前端实施要点

1. **页面风格**: 参照supplier/index.vue的实现，保持一致的UI风格
2. **API封装**: 参照@/api/system/supplier.js的格式封装API
3. **富文本编辑器**: 安装并集成wangeditor
4. **字典管理**: 添加商品状态等字典数据
5. **权限指令**: 使用v-hasPermi指令控制按钮显示

### 菜单配置

需要在sys_menu表中插入以下菜单数据：

- 商品管理目录 (parent_id=0)
- 仓库管理菜单 (parent_id=商品管理目录ID)
- 商品管理菜单 (parent_id=商品管理目录ID)
- 对应的按钮权限记录

## 页面规划

### 1. 仓库管理页面 (wms/warehouse/index.vue)

- **搜索区域**: 仓库名称、仓库编码、省份、状态
- **操作按钮**: 新增、删除、导出
- **数据表格**: 仓库名称、编码、省份、地址、类型、状态、备注、操作
- **新增/修改弹窗**: 表单布局，包含仓库名称、编码、省份选择器、地址、类型选择器、状态开关、备注

### 2. 商品管理页面 (wms/product/index.vue)

- **搜索区域**: 商品名称、SKU编码、状态
- **操作按钮**: 新增、删除、导出
- **数据表格**: 商品名称、SKU编码、预览图（缩略图）、售价、进货价、单位、状态、操作
- **新增/修改弹窗**: 
- 基本信息区域: 商品名称、SKU编码、简介、预览图上传
- 价格信息区域: 售价、进货价、单位
- 宣传内容区域: 移动端宣传页富文本编辑器
- 供应商信息区域: 供应商选择、供应商商品编码、报价、交货周期（可添加多个）
- 库存信息区域: 仓库选择、库存数量、预警数量（可添加多个仓库）
- 其他: 状态、备注

## 交互设计

- 表格支持多选、分页、排序
- 弹窗表单支持表单验证
- 图片上传支持预览和删除
- 富文本编辑器支持全屏编辑
- 供应商和库存信息使用动态表单，支持添加/删除多条记录
